## Project Status

As of 2026-05-12, the TensorRT project is functionally complete for the current scope:

- PyTorch baseline inference runs successfully
- ONNX export runs successfully
- ONNX export now defaults to dynamic batch
- TensorRT FP32 and FP16 engines build successfully
- ONNX Runtime inference runs successfully on multiple images
- TensorRT Python inference runs successfully on multiple images
- `trtexec` benchmark runs successfully
- Output validation and result collection run successfully

## Key Code Changes

- `scripts/export_onnx.py`
  - Default export is now dynamic batch
  - Added `--static-batch` to force fixed-batch export when needed

- `scripts/build_and_benchmark.py`
  - Detects static vs dynamic ONNX input shapes
  - Avoids passing invalid `--shapes` to static ONNX builds
  - Uses `--minShapes/--optShapes/--maxShapes` for dynamic ONNX models

- `scripts/run_onnx_infer.py`
  - Supports static batch-1 models by automatically splitting multi-image input into per-image runs
  - Still supports direct batched inference for dynamic-batch ONNX models

- `scripts/run_trt_infer.py`
  - Supports engines that only accept batch 1 by automatically splitting multi-image input
  - Saves engine input shape and session run count into benchmark metadata

- `README.md`
  - Updated to reflect dynamic ONNX export as the default path
  - Added rerun instructions after ONNX changes
  - Clarified TensorRT build/profile behavior and inference behavior

## Important Notes

- `models/model.onnx` is now dynamic batch
- Any time ONNX is re-exported, both TensorRT engines should be rebuilt
- Current TensorRT Python inference path supports multi-image input even when the engine profile is fixed at batch 1 by splitting the batch internally
- `trtexec` benchmark numbers are not directly comparable to TensorRT Python benchmark numbers because they measure different runtime paths
